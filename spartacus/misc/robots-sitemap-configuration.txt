REFERENCE: https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag

In /pages API n/w response we get a key `robotTag`.
Eg:
{ robotTag: "NOINDEX_NOFOLLOW" }
This helps add robots <meta> tag to <head> of our page.
Eg:
<meta name="robots" content="NOINDEX, NOFOLLOW">
noindex
Do not show this page, media, or resource in search results.
If you don't specify this rule, the page, media, or resource may be indexed and shown in search results.
nofollow
Do not follow the links on this page.
If you don't specify this rule, Search Engine may use the links on the page to discover those linked pages.

NOTE: IF BACKEND IS UNABLE TO CHANGE `/pages` APIs `robotTag` KEY, USE CUSTOM RESOLVER IN FRONTEND TO OVERRIDE
0.A. CUSTOM-BasePageMetaResolver.TS
=================================================
import {BasePageMetaResolver, CmsService, PageLinkService, PageRobotsMeta, RoutingPageMetaResolver, TranslationService} from '@spartacus/core';
import { environment } from '../../../environments/environment';
@Injectable({providedIn: 'root'})
export class CUSTOM-BasePageMetaResolver extends BasePageMetaResolver {
  constructor(
    protected override cmsService: CmsService,
    protected override translation: TranslationService,
    protected override routingPageMetaResolver: RoutingPageMetaResolver,
    protected override router: Router,
    protected override pageLinkService: PageLinkService
  ) {
    super(cmsService, translation, routingPageMetaResolver, router, pageLinkService);
  }
  override resolveRobots(): Observable<PageRobotsMeta[]> {
    return this.page$.pipe(
      switchMap((page) => {
        if (environment.production && page.template === 'ProductListPageTemplate') {
          return of([PageRobotsMeta.FOLLOW, PageRobotsMeta.INDEX]);
        }
        return super.resolveRobots();
      })
    );
  }
}

1.A. UPLOAD STATIC FILES TO CCv2
========================================
REFERENCE: https://help.sap.com/docs/SAP_COMMERCE_CLOUD_PUBLIC_CLOUD/0fa6bcf4736c46f78c248512391eb467/f5f8f622e9ac4f49a04cada647cdcb92.html?locale=en-US
           https://help.sap.com/docs/SAP_COMMERCE_CLOUD_PUBLIC_CLOUD/0fa6bcf4736c46f78c248512391eb467/2c29807b8f6a4115a3117fa6088199ed.html?locale=en-US
- Create a single folder containing all static files and ZIP it, so that it is less than 1 MB in size.
  This includes maintenancePage.html(s), robots.txt, sitemap.xml   etc...
- On CCv2, from the navigation menu, select `Static Files`  >  `Create`
  Add name, description, browse & upload .ZIP
- To add this static file set to an endpoint of an environment, select `Environments` from navigation menu.
- Select an environment  >  `Public Endpoints` panel
  click the name of an endpoint in that environment  eg: `YOUR-APP-SITE`
- Scroll to `Static Files` panel and
  select a static file set to apply from the option(s), SAVE your changes.
- To check where a static file set is applied,
  open the `Static Files` page from the navigation menu and find the column for Connected Endpoints.
  the label in that column identifies the `<environment_name:endpoint_name>` where the static file set was applied.
Eg:
robots.txt         > https://YOUR-APP-SITE/robots.txt 
error/503.html     > https://YOUR-APP-SITE/error/503.html
static/health.html > https://YOUR-APP-SITE/static/health.html
